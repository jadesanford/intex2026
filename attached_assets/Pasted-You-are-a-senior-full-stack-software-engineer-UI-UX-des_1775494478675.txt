You are a senior full-stack software engineer, UI/UX designer, and secure systems architect.

Your task is to design and scaffold a polished, production-style web application called **Open Arms**.

## Project Overview

Open Arms is a nonprofit-style organization presented as serving girls who are survivors of sexual abuse and trafficking. The public-facing brand is positioned for **Indonesia**, with a gentle, trustworthy, and professional tone. The purpose of the web app is to support:

1. survivors seeking help,
2. donors wanting to support the mission,
3. staff managing safehouse operations and case records,
4. leadership using analytics to make better decisions.

This is a class project for **IS 413 – Enterprise Application Development**.

## Core Design Direction

The website should feel:

* **professional**
* **safe and soft for girls**
* **modern nonprofit**
* **hopeful, calm, and welcoming**
* not cold, not corporate-heavy, not overly clinical

Use:

* **flat modern nonprofit illustrations**
* **rounded cards**
* **soft shadows**
* clean spacing
* a polished and consistent design system

### Branding

* Organization name: **Open Arms**
* Tagline: **A safe place to heal**
* Public-facing audience: girls needing help, donors, supporters, and partners
* Tone: compassionate, respectful, calming, trustworthy

### Religious Theme

Include a **gentle “Faith & Hope” section** on the public-facing site only.
The Christian theme should be **subtle and contained**, not dominant across the whole product.
Use themes of:

* love
* hope
* healing
* open arms
* compassion

Do **not** make the site look like a church website. It should still feel like a polished nonprofit organization.

## Tech Stack Requirements

Use:

* **Backend:** .NET 10 / ASP.NET Core Web API / C#
* **Frontend:** React + TypeScript + Vite
* **Database:** PostgreSQL preferred
* **Authentication:** ASP.NET Identity

* **Charts:** modern charting library suitable for React

## Security and Privacy Requirements

Because the system contains highly sensitive data involving minors and abuse survivors, prioritize privacy and security in all design choices.

Implement:

* HTTPS enforcement
* HTTP to HTTPS redirect
* secure password policy stronger than default ASP.NET Identity settings
* RBAC with roles such as Admin, Staff, Donor
* authenticated and authorized API access
* restricted access to sensitive resident data
* delete confirmation flows
* input validation and sanitization
* CSP header
* HSTS if possible
* secure handling of secrets using environment variables or secrets manager
* safe handling of cookies
* no sensitive data exposed publicly

Sensitive notes fields should be protected and only shown to authorized users.
